# `/deepen` ワークフロー

このワークフローは、複雑な課題に対して性急な実装を避け、議論を通じて設計を深化させ、確実な知見を蓄積するためのモードです。

## 実行ステップ

### 1. 「設計深化モード」の宣言
エージェントは、現在のセッションが設計深化フェーズであることを明示的に定義します。
> 🤖 **設計深化モードを開始します。思索と議論を最優先し、現時点での実装は行いません。**

### 2. 文脈の再整理と論点の詳細化
- `docs/designs/` 等において、解決すべき課題の**技術的・論理的背景**を詳細に記述します。
- メリット・デメリット、代替案との比較、将来的な拡張性への影響など、多角的な論点を抽出します。
- 必要に応じて実験を行い、事実に基づいた深い議論の材料を揃えます。

### 3. 実装手順の完全決定（トライアンドエラーの排除）
- ADR (`docs/decision_log.md`) において「なぜこの道を選んだか」だけでなく、他案の棄却理由を明確にします。
- **実装フェーズでの試行錯誤を禁止**するため、使用する関数名、データ構造、異常系処理を論理的に確定させます。
- 不具合修正の場合、現象の再現条件と根本原因を特定し、副作用の可能性を全てこのフェーズで議論し尽くします。

### 4. 知見と議論の完全な蓄積
- ユーザーから `/approve` を得るまで、議論の「飽和（これ以上論点が出ない状態）」を目指して深掘りを継続します。

## 実装モードへの移行条件
- ユーザーから設計に対する**明示的な承認（OK / Approve 等）**を得る。
- 承認後、エージェントは「実装モードへの昇格」を宣言し、通常の開発フロー（実装ツールの解禁）に移行します。
